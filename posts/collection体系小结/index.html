Collection体系小结

在编程时候，我们需要集中存放多个数据，数组是个很好的方法，但是有时候数组不能很好的完成一些事情，这时候就需要我们用集合。集合跟数组相比有以下几个特点：
1. 长度可以改变
2. 只能存储对象
3. 可以存储多种类型对象

Collection

Collection是最基本的集合接口，一个Collection代表一组Object，即Collection的元素（Elements）。一些Collection允许相同的元素而另一些不行。一些能排序而另一些不行，于是衍生出两个子类接口List和Set。

collection体系的结构图

Collection是一个接口，该接口定义了如下操作集合元素的方法：

  1. boolean add(Object o):向集合添加一个元素。
  2. boolean addAll(Collection c):把集合C中的所有元素添加到另一个集合中。
  3. Void clear():删除集合中所有元素。
  4. boolean remove(Object o):删除集合中指定元素o。
  5. boolean removeAll(Collection c):从集合中删除集合C里包含的所有元素。
  6. boolean isEmpty():判断集合是否为空。
  7. boolean contains(Object o):判断集合中是否存在指定元素o。
  8. boolean cotainsAll(Collection c):判断集合中是否包含集合C中的所有元素。
  9. lterator iterator():返回一个lterator对象，用来遍历集合中的元素。
  10. int size():返回集合中的元素个数。
  11. boolean retainAll(Collection c):求交集的功能，从集合中删除集合C里不包含的元素。
  12. Object[] toArray():把集合变成数组。

List

List最常用的是类是ArrayList,其本质上就是一个数组。数组长度是有限的，ArrayList可无限"存储”,这其中就是动态扩容实现的。动态扩容简单来说就是创建一个更大的空间，把所有数据拷贝进去。

Set

Set接口下的元素是无序且不可重复的。

HashSet

利用Hash算法实现的Set,保证其中元素的唯一性依据两个重要的约定：equals与hashCode

  1. 首先判断hashCode()值是否相同
  2. 相同再继续判断equals(),看返回值，true就添加进集合

hashCode有三大原则：

  1. 同一对象必须始终返回相同的hashCode值
  2. 两个相同的equals返回true,必须返回相同的hashCode值
  3. 两个对象不等，也可能返回相同的hashCode值

Map

Map是不同于collection的存在，map是一个键值对形式的集合。它的元素都是有键和值组成。Map的键(key)是唯一的,值(value)可以重复。

Map结构图

Map接口定义了如下方法：

  1. V put(K key ,V value) :当key在集合中不存在是,添加元素;当key存在时替换元素
  2. boolean containsKey (Object key) :判断指定的键是否在集合中存在
  3. boolean containsValue(Object value):判断指定的值是否在集合中存在
  4. boolean isEmpty() :判断集合是否为空
  5. Void clear():清除所有键值对数据
  6. Object get (Object key) :根据键获取值
  7. Set keyset(): 所有键的集合
  8. Collectionvalues() :所有值的集合
  9. Int size()：返回Map里的key-value对的个数

HashMap与HashSet本质上是一种东西
